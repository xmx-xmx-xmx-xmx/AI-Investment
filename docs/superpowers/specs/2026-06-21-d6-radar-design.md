## 概述

飞书新建「雷达观测表」+ `src/radar.py` 模块。对标的高波动卫星标的（ETF/个股/基金）做双向信号检测（抄底 + 追涨），每日早间/收盘前简报注入信号，LLM 轻度确认。

核心原则：系统算数，你做判断。不做迷你雪球，不管理持仓，不输出操作指令——只告诉你「这个标的值得看一眼了」。

## 一、飞书「雷达观测表」

新增一张飞书多维表格，字段设计：

| 字段 | 类型 | 填写方 | 说明 |
|------|------|--------|------|
| 标的代码 | 文本 | 你手工填 | `515080` / `QQQ` / `00700` / `MU` / `03076` |
| 标的名称 | 文本 | 你手工填 | 「中证红利ETF」「纳斯达克100」 |

| 趋势 | 文本 | 系统自动 | 复用 `price_updater.py` 趋势检测（右侧企稳/左侧下跌/横盘震荡） |

| 入库日期 | 日期 | 系统自动 | 首次 `scan_radar()` 执行时自动填入当天日期 |
| 状态 | 单选 | 系统自动 | 观察中 / 有信号 / 已入场 / 已放弃 |

## 二、信号算式（`src/radar.py`）

### 2.1 抄底信号（双窗口 + 双档位）

抄底 = 一定跌幅 AND 趋势企稳。窗口和阈值：

| 信号 | 窗口 | 条件 | 趋势要求 |
|------|------|------|----------|
| 🟡 关注 | 10 日跌幅 ≤ -5% | 轻度低估 | 趋势 = "右侧企稳" |
| 🔵 底部反转 | 20 日跌幅 ≤ -8% | 深跌修复 | 趋势 = "右侧企稳" |

两档独立判定，可能同时命中。任一命中 + 趋势满足 = 亮灯。

### 2.2 追涨信号 🟢

追涨 = 连续上涨 AND 仍未溢价：

- 近 5 日每日 `change_pct > 0`（连续阳线）
- 现价 ≤ 20 日均线 × 1.03（刚突破、没飞）

两者同时满足 = 🟢 趋势加速。

### 2.3 数据不足策略

- 入库不足 5 个交易日：追涨信号空白
- 入库不足 10 个交易日：🟡 关注空白
- 入库不足 20 个交易日：🔵 底部反转空白
- 均线计算用已有数据天数（不强行要求满 20 日）

### 2.4 状态流转

```
观察中 ──信号命中──→ 有信号 ──你手动移入底仓表──→ 已入场
  │                      │
  └──── 你手动放弃 ──────→ 已放弃
```

`src/radar.py` 只负责系统字段，状态变更由你手动在飞书表操作。

## 三、简报注入（LLM 轻度确认）

### 3.1 早间 `morning`

在「隔夜要闻」之后、「AI 解读」之前插入雷达区块。只有当有信号时才走 LLM：

```
🔭 雷达扫描（2 有信号 / 共 8）

· MU (美光科技)
  🟢 趋势加速（5日阳线）| 现 $98.50 | 关联: 03076
  🤖 隔夜存储芯片板块利好，追涨信号有基本面支撑

· 159509 (纳指科技ETF)
  🟡 关注（10日 -6.2%）| 现 ¥1.52 | 关联: (无)
  🤖 纳指连续修复但VIX仍在20+，反弹偏弱，可再观察
```

无信号时只产出 1 行「雷达扫描：今日无新增信号」，不走 LLM。

LLM Prompt 输入：信号标的列表 + 当天要闻标题 + 宏观日历摘要。输出：每只 1 句确认。

### 3.2 收盘前 `closing`

使用当日最新行情重算，插入「Python 策略中枢」上方。格式同早间。

### 3.3 其他时段不注入

`midday` / `sat_morning` / `sun_evening` 不做雷达扫描——早间给认知、收盘前给确认即可。

### 趋势检测适配

`price_updater.detect_fund_trend()` 目前依赖 akshare 返回的 DataFrame。雷达标的多为 ETF/个股，不能用同一个函数。在 `radar.py` 中实现轻量版：

```python
def _detect_trend(prices_5d: list[float]) -> str:
    """用最近 5 日收盘价判断趋势方向。"""
    if len(prices_5d) < 5:
        return ""
    last_3 = prices_5d[-3:]
    if all(last_3[i] < last_3[i + 1] for i in range(2)):
        return "右侧企稳"
    if prices_5d[-1] < prices_5d[0]:
        return "左侧下跌"
    return "横盘震荡"
```

输入 5 日收盘价列表，输出趋势标签，与现有系统标签一致。
